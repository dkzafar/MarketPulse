We need to guarantee that our end users always see **every** asset we’ve ingested—no matter how many there are. Implement these changes in our data-fetching pipeline **and** our front-end listing component:

1. **Back-End: True Multi-Page Fetching**  
   a. For each free‐API source (AlphaVantage, TwelveData, Finnhub, Yahoo Finance, CoinGecko, etc.), detect and respect its pagination scheme:  
      - If the API returns a `nextPageToken` or `offset`/`limit` fields, loop through them until no more pages remain.  
      - Aggregate **all** pages into our staging table before dedupe/validation.  
   b. After *every* full run, compare the count of `master_assets` to the count in the staging table; if they differ, log which page(s) failed and retry them with exponential back-off.  
   c. Expose an endpoint `GET /assets?all=true` that always returns the full, de-duplicated list from your canonical production table.

2. **Front-End: Infinite Scroll / Full Pagination**  
   a. Remove any hard-coded “show first 96” or default `limit=96`.  
   b. Implement either:  
      - **Infinite Scroll**: As the user scrolls down the list, automatically call `GET /assets?offset=<current>&limit=<pageSize>` and append to the DOM until no more assets remain.  
      - **Explicit Pagination**: Show page controls (‹ Prev / Next ›) that iterate through every chunk of your full list.  
   c. Display a “Showing X of Y assets” header so users know the total.  
   d. Ensure the UI calls `GET /assets?all=true` (or loops through pages) on initial load if you’re not paginating—then render them all.

3. **Automated Verification**  
   a. After your next full back-end run, compare the count in `master_assets` vs. what your UI is actually rendering (via a headless-browser test).  
   b. If UI_count < master_assets_count, fail the check and send an alert (Slack/email) that “UI is only showing N of M assets.”  

4. **Performance & UX**  
   a. Lazy-load images/charts for off-screen tiles.  
   b. Use a lightweight placeholder (skeleton cards) while data streaming is in progress.  

5. **Monitoring**  
   a. Track metrics:  
      - **Back-end**: total assets ingested, pages fetched, retry counts.  
      - **Front-end**: total assets rendered, time to first paint, scroll-through completion rate.  
   b. Alert if rendered assets ≠ ingested assets for more than 5 minutes.

By following these steps, your UI will no longer be capped at 96 or 130—it will faithfully show the entire universe of assets you’ve fetched from all pages of every free API, matching (or even exceeding) the breadth of eToro’s catalogue.
